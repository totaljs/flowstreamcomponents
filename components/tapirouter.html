This component routes incoming Total.js API data according to the defined API schema. The component extends input data (query, payload and params) with parsed data. __Input & Output data__: ```js { // REQUIRED payload: payload: { schema: 'schema_name/{param_1}/{param_2}?query=arguments', data: {} // optional, request payload }, ip: String, // optional, IP address url: String, // optional language: String, // optional ua: String, // optional, parsed UA user: Object, // optional headers: { key: String }, // optional, request headers cookies: { key: String }, // optional, cookies query: { key: String }, // optional params: { key: String }, // optional files: [{ type: String, filename: String, path: String, ext: String, size: Number, width: Number, height: Number }] // optional } ``` In the case of `error`, the input data is extended by the `response` property with an error.
$NAME
Every name of identifier must be unique per Flow. You can use dynamic arguments in the form {argument_name}
Use names in lower-case. We recommend to keep e.g. users_read/{id} or products_read/{id}
